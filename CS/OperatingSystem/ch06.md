# Chapter 6: 동기화
#운영체제

## 백그라운드
* 프로세스를 동시에 실행할 수 있음
	* 언제든지 중단될 수 있으며, 부분적으로 실행을 완료할 수 있음
* 공유 데이터에 동시에 액세스하면 데이터 불일치가 발생할 수 있음
* 데이터 일관성을 유지하기 위해서는 협력 프로세스의 질서 있는 실행을 보장하는 메커니즘이 필요

## Producer
```c
while (true) {
    while (counter == BUFFER_SIZE);

    buffer[in] = next_produced;
    in = (in + 1) % BUFFER_SIZE;
    counter++;
}
```

## Consumer
```c
while (true) {
    while (counter == 0);
    
    next_consumed = buffer[out];
    out = (out + 1) % BUFFER_SIZE;
    counter--;
}
```

## Race Condition
* `counter++` 은 다음과 같이 구현될 수 있음
```
register1 = counter
register1 = register1 + 1
counter = register1
```
* `counter--` 은 다음과 같이 구현될 수 있음 
```
register2 = counter
register2 = register2 - 1
counter = register2
```
* 문제점 발생
	* S0: producer 실행 register1 = counter
	* S1: producer 실행 register1 = register1 + 1
	* S2: consumer 실행 register2 = counter
	* S3: consumer 실행 register2 = register2 - 1
	* S4: producer 실행 counter = register1
	* S5: consumer 실행 counter = register2

## Critical Section 문제
* {p0, p1, …} 프로세스의 시스템을 고려
* 각 프로세스에는 코드의 *중요한 섹션* 세그먼트가 있음
	* 프로세스는 공통 변수 변경, 테이블 업데이트, 파일 쓰기 등을 수행할 수 있음
	* 한 프로세스가 중요한 섹션에 있을 때 다른 프로세스가 중요한 섹션에 있지 않을 수 있음
* 중요한 섹션 문제는 이 문제를 해결하기 위한 프로토콜을 설계하는 것
* 각 프로세스는 *입력 섹션*의 중요 섹션 진입 허가를 요청해야 하며, *종료 섹션*이 있는 중요 섹션 다음에 *나머지 섹션*으로 이동할 수 있음

## Critical Section
```c
do {
    while (turn == j);  // entry section
        /* critical section */
    turn = j;  // exit section
        /* remainder section */
} while (true);
```

## Critical Section 문제 해결책
1. Mutual Exclusion: 프로세스 Pi가 중요한 섹션에서 실행 중인 경우, 다른 프로세스가 중요한 섹션에서 실행될 두 없음
2. Pregress: 핵심 섹션에서 실행 중인 프로세스가 없고 해당 핵심 섹션에 진입하려는 프로세스가 있는 경우, 다음 중요 섹션에 진입할 프로세스의 선택을 무기한 연기할 수 없음
3. Bounded Waiting: 프로세스가 중요한 섹션으로 진입하도록 요청한 후 해당 요청이 허용되기 전에 다른 프로세스가 중요한 섹션으로 진입할 수 있는 횟수에 제한이 존재해야 함
	* 각 프로세스가 0이 아닌 속도로 실행된다고 가정
	* n 프로세스의 상대 속도에 관한 가정 없음

## Critical Section Handling
* kernel이 preemptive인지 non-preemptive인지에 따라 두 가지 접근 방식
	* *Preemptive*: kernel 모드에서 실행할 때 프로세스의 preemption을 허용
	* *Non-preemptive*: kernel 모드를 종료하거나, CPU를 차단하거나, CPU를 자발적으로 산출할 때까지 실행
		* kernel 모드에서 기본적으로 레이스 조건이 없음

## Peterson’s Solution
* 문제 해결에 대한 좋은 알고리즘
* 이중 프로세스 솔루션
* load 및 store 시스템 언어 명령이 원자적이라고 가정(중단될 수 없음)
* 두 프로세스는 두 가지 변수를 공유
	* `int turn;`
	* `Boolean flag[2]`
* 변수 `turn`은 크리티컬 섹션에 들어갈 차례
* `flag` 배열은 프로세스가 크리티컬 섹션에 진입할 준비가 되었는지 나타내는데 사용
	* `flag[i] = true`는 프로세스 Pi가 준비되었음을 의미

```c
do {
    flag[i] = true;
    turn = j;
    while (flag[j] && turn == j);
        /* critical section */
    flag[i] = false;
        /* remainder section */
} while (true);
```
