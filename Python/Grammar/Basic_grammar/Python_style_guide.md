#### 이름 규칙

- 모든 변수와 함수는 **소문자**로 작성
- 여러 단어일 경우 **언더바(_)** 로 구분
- 상수는 **대문자**로 작성

#### 의미있는 이름

- **함수명**과 **변수명**만 보고도 **어떤 역할을 하는지 유추할 수 있는 의미 있는 이름**으로 짓기

#### 들여쓰기

- **Tab키** 사용 or **스페이스 4개** 사용

#### 함수 정의

- 함수를 정의할 때 **위 아래로 빈 줄이 두개**씩 있어야 함
- 파일의 **첫 줄이 함수 정의인 경우, 함수 위에는 빈줄 생략**

#### 괄호 안

- **괄호 바로 안**에는 띄어쓰기를 안함

#### 함수 괄호

- 함수를 정의하거나 호출할 때, **함수 이름과 괄호 사이**에 띄어쓰기를 안함

#### 쉼표(,)

- **쉼표 앞**에는 띄어쓰기를 안함

#### 지정 연산자

- 지정 연산자 앞, 뒤로 띄어쓰기 하나씩

#### 연산자

- 기본적으로 연산자 앞, 뒤로 띄어쓰기 하나씩
- 연산의 **우선순위를 강조**하기 위해서는 생략 가능

#### 코멘트(#)

- 일반 코드와 같은 줄에 코멘트를 쓸 경우, 코멘트 앞에 **띄어쓰기 최소 2개**
